指令集结构(Instruction Set Architecture,ISA)
是软硬件的接口

应用程序
操作系统
↓
指令集
↓
CPU
一般分为两类:
  • 复杂指令集计算机CISC(Complex Instruction Set Computer)
  • 精简指令集计算机RISC(Reduce Instruction Set Computer)
    MIPS （ Microprocessor without Interlocked Pipeline Stages，无内部互锁流水线级微处理器） 指令集
    • 属于精简指令集（RISC）类型
    每条机器指令是一个二进制串
    操作码OP 操作数A
    add(助记符) $(寄存器名)

寄存器(Register)
通用寄存器(General purpose register,GPR)

  • 微处理器内部
  • 工作速度与CU和ALU相同
  • MIPS 32指令集中,定义了32个32位的通用寄存器,所以编号用5个bit

    特殊寄存器
  • 标识CPU状态的
  • 控制CPU某些功能部件(Cache等)
  • 暂存某些临时中间变量结果的(专用指令读取)
    为了支持浮点数运算,还有专门的浮点寄存器

寻址方式(Addressing mode)
定义:指微处理器根据指令中的操作数部分给出的信息定位所需指令或所需数据
存储器的编址单元

  • MIPS定义了存储器的半字(half word)和字(word)分别是2字节,和字字节
  • 地址对齐


存储器的寻址方式

  • 直接寻址
    • 指令直接给出操作数的内存地址A,操作数在A指向的存储单元中
      操作码OP 有效地址A
  • 间接寻址
    操作码OP 寄存器号R
  • 优势在于R占位少
  • 相对寻址,有个漂移量
    其他寻址方式
  • 寄存器寻址
  • 直接寻址

MIPS指令集结构
MIPS 32指令格式

  • 指令32位宽(字长),按字地址对齐(字地址为4的倍数)
    • 三种指令格式,组成结构要求记牢
      • R型指令:两个操作数和结果都在寄存器中的运算指令
        • op:全是0,具体功能由func确定
        • 三个寄存器操作数:(r是寄存器register,t是target,s是source,d是destination)
          • rs:第一个源操作寄存器
          • rt:第二个源操作寄存器
          • rd:目的操作寄存器
        • sa:
          • 对非移位指令,为00000
          • 对移位指令,为移位次数
            op rs rt rd shamt func
            6bits 5bits 5 5 5 6

  • I型指令:包含立即数的指令
    • op:确定指令的功能
    • rs:第一个源操作数.是寄存器操作数,或偏移寻址中的基地址寄存器(存取指令)
    • rt:目的寄存器
    • immediate:立即数,根据指令的不同,可能是第二个源操作数,或和rs组成偏移寻址方式
  • 此处补充了一个可能的考点:LOAD,STROGE,在STROGE中,rt是源操作数,因为方向是把数据存储从寄存器放到存储器中,所以要反过来
  • 还讲了地址的低两位为0,这样可以确保两个地址之间的字节数都是4的整数倍
    op rs rt immediate
    6bits 5bits 5bits 16bits
    直接看文档吧

MIPS地址空间分配
有个逻辑地址和物理地址的概念,区别在于逻辑地址是程序员可见的,物理地址根本看不到
他们之间有个地址映射,具体计算我们也并不关心
逻辑地址是指的CPU分配给每个进程的地址,一个进程在内存中的实际地址和进程认为它所处的地址是不同的

异常与中断
异常:内部出的差错,溢出,除零…
中断:外部的打断,断电,打印机缺纸…
发生异常或中断时,CPU会切换到内核态,使用Cause寄存器记录异常原因
