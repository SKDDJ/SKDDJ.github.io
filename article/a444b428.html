    排序算法的衡量标准:
  • 时间效率 —比较次数与移动次数
  • 空间效率
  • 稳定性(如果关键字A和B相等,那么排序前后AB的相对顺序应该不变)

插入排序

  • 顺序表中的第一个元素位置留作哨兵,用于比较时存放元素
    时间上的分析: 如果有n个元素,那么进行n-1次排序就可以排好序
    每次都把待排序对象中的一个元素插入到已经排好序的数组(有序子序列)中,插入完(先跟有序子序列的最后一个元素比较可以省事) 循环完也就排序完了
    在插入比较时,可以采用:
    直接插入排序
  • (挨个遍历)平均O(n^2),最好O(n){只需比较不用移动}

    void InsertSort(SqList&L){

    int i,j;

    for(i=2;i<=L.length;i++)

    if(L.r[i].key<L.r[i-1].key) //将L.r[i]插入有序子表

    {L.r[0] = L.r[i];//复制为哨兵

    L.r[i]=L.r[i-1];

    for(j=i-2;L.r[0].key<L.r[j].key;j--)

    L.r[j+1]=L.r[j];//记录后移

    L.r[j+1]=L.r[0];//插入到正确位置

    }

    }

希尔排序
基于直接插入排序,每一次操作是先对每隔一个增量(这个增量会逐渐减少到1)构成一个子表,然后对每个子表进行直接插入排序,坏处是每次都有跳跃(除了最后一次),稳定性不行

交换排序
思想:如果发生逆序,就交换,直到排好序
冒泡排序O(n^2)

  • 也是n个元素,n-1次循环比较就够了(最后一次不用比,只剩一个最大或最小了)
  • 正着或倒着两两比较,遇到逆序就交换,好处是每次排序都会从剩下没排序的部分挑出最大值或者最小值

    void BubbleSort(ElemTyep A[], int n){

    for(i = 0 ; i< n -1;i++)

    flag=flase;//本趟冒泡是否交换的标志

    for(j = n - 1; j > i; j--)//从最后一个开始倒着比较

    if(A[j-1]>A[j]){//如果发生前面的元素比后面的大

    swap(A[j-1],A[j]);

    flag = true;//发生交换flag记为true

    }

    if(flag == false)//如果没有交换,flag==false,那么这个表就是有序的,可以退出了

    return;

    }

    **快速排序O(nlogn)**
  • low和high两个指针从两头向中间逼近
  • 递归算法
  • 所有内部排序算法中平均性能最优的算法
  • ⚠️,不适合用于原本有序或者基本有序的序列

简单选择排序

  • 每次都是把最末尾开始倒着数最小的和前面的大的做交换,这样小的就像是有序了不用考虑
  • 一样n个元素,n-1趟排序就够了
  • 也不稳定,因为有跳动

    **堆排序** 构建大根堆(就是根一定要是最大的)
  • O(nlogn)
  • 不稳定

    每次都是找出最大的那个数值然后从堆里移除
    补充下满二叉树和完全二叉树的概念:
  • 完全二叉树:相较于满二叉树,最后一层的叶节点不一定全满
