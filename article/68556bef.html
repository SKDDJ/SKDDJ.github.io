协程(goroutine)
现在CPU很多个核,一个核也基本支持多线程,可以同时执行多条指令流水线,因此有了并发(concurrent)计算.
协程又名*绿色线程.*由程序的运行时(runtime)维护的线程.Go不支持创建系统线程,所以协程是Go程序内部并发唯一实现方式
每个Go程序启动时只有一个对用户可见的协程,称为主协程.

在Go中,开启一个新协程很简单,只需要在一个函数调用之前使用一个​go​关键字,就可以让该函数运行在一个新的协程中.当函数调用退出后,这个新的协程也随之结束.我们将该函数调用称一个协程调用(或者此协程的启动调用).一个协程调用的所有返回值都必须全部舍弃.
当一个程序的主协程退出后,此程序也就退出了,无论是否还有其他协程在运行.

并发同步(concurrency synchronization)
关于数据竞争(data race)

  • 在一个计算向一段内存写数据时,另一个计算从内存中读数据,结果导致读出的数据完整性得不到保证
  • 在一个计算向一段内存写数据时,另一个计算也向此段内存写数据,结果导致被写入的数据完整性得不到保证
    并发编程还包括的任务有:
  • 决定需要开启多少计算
  • 决定何时开启,阻塞,解除阻塞和结束哪些计算
  • 决定如何在不同的计算中分担工作负载
    Go支持几种并发同步技术,其中通道是最独特和最常用的
    还可以利用​sync​标准库包中的​WaitGroup​来同步主协程和协程
    WaitGroup有三个方法:
  • Add注册新需要的任务数
  • Done通知某个任务已经完成了
  • Wait阻塞到所有任务完成后再继续执行后面的语句

协程的状态
一个活动中的协程可以有两个状态:运行状态和阻塞状态(像是把协程冻住了一样😄)
注意:一个处于睡眠中的(通过time.sleep)或者等待系统调用返回的协程被认为处于***运行状态***
死锁:当一个运行中的程序中的所有协程都处于阻塞状态,编译器会让这个程序崩溃的
然后就可以利用这些知识写一个死锁程序了:

package main

import (

"sync"

"time"

)

var wg sync.WaitGroup

func main() {

wg.Add(1)

go func() {

time.Sleep(time.Second * 2)

wg.Wait()

}()

wg.Wait()

}

协程的调度
在任意时刻,最多只能有和逻辑CPU数目一样多的协程在同时执行

  • 可以使用​runtime.NumCPU​函数来查询可用逻辑CPU数目
    • 每个逻辑CPU在同一时刻最多执行一个协程
      • 一个协程在同一时刻只能在一个系统线程中执行
      • 所以Go运行时(runtime)必须让逻辑cpu频繁的在 不同的 处于 运行状态的 协程之间切换
      • ps 这就和操作系统执行系统线程的原理是一样的.
        协程的生命周期:
  • 注意⚠️ : 睡眠和等待系统调用返回子状态被认为是运行状态,而不是阻塞状态
    标准编译器采纳了一种被称为M-P-G模型的算法来实现协程调度.
  • M:表示系统线程
  • P:逻辑处理器
  • G:协程
    • 逻辑处理器(P)把 不同的 处于运行状态协程(G) 交给 不同的 系统线程(M)
      在运行时刻,我们可以使用​runtime.GOMAXPROCS​函数来获取和设置逻辑处理器的数量
  • Go1.5后默认逻辑处理器和逻辑CPU数量一致
    • 但是在某些文件操作很频繁的程序中,设置一个大于​runtime.NumCPU()​的​GOMAXPROCS​值可能是有好处的
  • 也可以通过设置​GOMAXPROCS​环境变量来设置一个Go程序的初始逻辑处理器数量

延迟函数调用(deferred function call)
​defer​关键字的作用
类似于栈一样的操作,把函数一直延迟执行,取出顺序是先进后出(如果有多个延迟操作)
和协程调用类似的是,被延迟的函数调用的所有返回值(如果存在)必须全部被舍弃
例子不再展示
一个延迟调用可以修改包含此延迟调用的最内层函数的返回值

package main

import "fmt"

func Triple(n int)(r int){

defer func(){

r += n

}()

return n + n

}

func main(){

fmt.Println(Triple(5))

}

- 返回15
  • 一个延迟调用的实参是在此调用对应的延迟调用语句被执行时被估值的.说人话就是defer作用就是把这个函数压入调用栈里面的时候就开始估值了,当从栈里面弹出时会使用压入时估的值.
  • 而一个匿名函数体内的表达式是在此函数被执行时才会被逐渐估值,不管这个函数被普通调用还是延迟/协程调用(特殊的匿名函数😄)

    package main

    import "fmt"

    func main() {

    func() {

    for i := 0; i < 3; i++ {

    defer fmt.Println("a:", i)

    }

    }()

    fmt.Println()

    func() {

    for i := 0; i < 3; i++ {

    defer func() {

    fmt.Println("b:", i)

    }()

    }

    }()

    }


为了让第二个循环输出b和第一个a相同,有两种手段(思想都是一样的,用一个形参代替循环变量实参i):

  • 给延迟调用的func传入i这个形参

    for i := 0; i < 3; i++ {

    defer func(i int) {

    // 此i为形参i，非实参循环变量i。

    fmt.Println("b:", i)

    }(i)

    }

  • 用新的i遮挡循环变量i

    for i := 0; i < 3; i++ {

    i := i // 在下面的调用中，左i遮挡了右i。

    // <=> var i = i

    defer func() {

    // 此i为上面的左i，非循环变量i。

    fmt.Println("b:", i)

    }()

    }

恐慌(panic)和恢复(recover)
Go不支持异常抛出和捕获,而是推荐使用返回值显式返回错误.
但是也有一套类似的机制(异常抛出/捕获),名叫—恐慌/恢复(panic/recover)机制
恐慌&恢复:

  • 如果一个函数调用产生了恐慌,那么立刻进入退出阶段
  • 在一个延迟函数调用之中调用内置函数​recover​,当前协程中的恐慌可以被消除
  • 协程在恐慌状态下退出,整个程序都会崩溃
    两个内置函数的声明原型:

    func panic(v interface{})

    func recover() interface{}

    接口(interface)类型类似于其他语言中的any或者Object类型,就是未定义的一个实参值,输入进去会自动判断类型 ​recover​函数返回的参数值是​panic​函数调用消费的参数 看这个用例就足够明白了:

    package main

    import "fmt"

    func main() {

    defer func() {

    fmt.Println("正常退出")

    }()

    fmt.Println("嗨！")

    defer func() {

    v := recover()

    fmt.Println("恐慌被恢复了：", v)

    }()

    panic("拜拜！") // 产生一个恐慌

    fmt.Println("执行不到这里")

    }

    ***输出结果:***

    嗨！

    恐慌被恢复了： 拜拜！

    正常退出

有些致命性错误不属于恐慌
对于官方标准编译器来说,栈溢出或内存不足不能被恢复,一产生就会崩溃
